# Code of conduct

## Expected conduct

Project participants must:

- discuss technical work respectfully and directly;
- critique code and evidence rather than people;
- provide reproducible details for bugs and reviews;
- respect privacy and never publish credentials or personal data;
- avoid harassment, discrimination, threats, and sexualized conduct; and
- preserve the project's public-data, no-signal, no-execution boundary.

## Scope

This code applies to repository issues, pull requests, reviews, discussions, and other public project spaces. It also applies when someone officially represents the project.

## Enforcement

Maintainers may edit or remove content, close or lock conversations, reject contributions, or restrict participation when conduct violates this policy. Enforcement decisions should be based on documented behavior and applied consistently.

Report abusive GitHub content using GitHub's reporting tools. Do not place sensitive personal information in a public issue. Security vulnerabilities must follow [SECURITY.md](SECURITY.md), not the public issue tracker.

## Attribution

This project-specific policy is informed by common open-source community standards and GitHub's Community Guidelines.
